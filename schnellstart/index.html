Eigene Installation

Sie möchten eine eigene Installation von SimpelMed nutzen?
Führen Sie folgende Schritte durch:

Voraussetzung:
Plattform GNU/Linux der Debian-Familie - Debian, Ubuntu, Mint usw.; Root-Rechte erforderlich.
Andere Betriebssysteme wie Windows oder RPM-Linuxe, MacOS, BSD etc. können per Docker oder Source-Code-Installationen betrieben werden. Die Bereitstellung der notwendigen Paktete erfolgt a.s.a.p.

→ Empfohlen ← Option 1, der elegante Weg »top-down«

a) Installation einer PostgreSQL-Datenbank:
sudo apt install postgresql postgresql-contrib
b) Einrichten der PostgreSQL-Datenbank/Zugänge,
d.h., /etc/postgresql/15/main/postgresql.conf und …/pg_hba.conf konfigurieren, siehe hier,
c) Installation der GNUmed-Palete:
sudo apt install gnumed-server gnumed-client gnumed-client-de
d) Einspielen der SimpelMed-Datenbank sowie
e) Modifikation auf SimpelMed.
Zu d) und e) siehe hier bzw. dort.

Mit dieser Option entfällt der zeitaufwändige und fehleranfällige GNUmed-Bootstrap-Prozess.

Option 2, der klassische Weg »bottom-up«

a) Installation einer PostgreSQL-Datenbank
b) Einrichten der PostgreSQL-Datenbank/Zugänge
c) Installation der GNUmed-Pakete
(bisher wie oben Option 1)
d) Bootstrap der GNUmed Inhalte
e) Modifikation auf SimpelMed, hier.

Mit Option 2 dauert es länger, der Anwender kann aber eigene Dateien einspielen.
